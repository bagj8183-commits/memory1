## I. 개요 및 적용 원칙
*   **목표:** 전반적인 브랜드 일관성 확보 및 사용자 경험(UX)의 예측 가능성을 높여 신뢰도 향상.

    *   Neutral-Dark: `#2C3E50` (본문 및 제목 텍스트)
    *   Background: `#FFFFFF` / Light Gray: `#F7F9FA`

## II. 핵심 컴포넌트 상세 스펙

### 1. CTA 버튼 (Call to Action Button)
가장 중요하며, 사용자의 행동을 유도하는 요소이므로 상태(State) 정의가 필수적입니다.

| 속성 | Primary (핵심 액션) | Secondary (보조/다운로드) | Tertiary (정보 보기) |
| :---: | :---: | :---: | :---: |
| **용도** | 결제, 시작하기, 주요 전환 유도 | 자료 다운로드, 약관 동의 등 보조 액션 | 더 알아보기, 상세보기 등 정보 제공 |
| **배경색 (Base)** | Primary Blue (`#1A4798`) | Transparent / Light Gray Background | Outline Only (Primary Border) |
| **텍스트 색상** | White (`#FFFFFF`) | Dark Gray Text (`#2C3E50`) | Primary Blue Text (`#1A4798`) |
| **Border Radius** | 6px | 6px | 1px |
| **Padding (H/V)** | 16px / 12px | 16px / 12px | 16px / 10px |
| **Hover State** | Background Darkening (`#14397a`) | Light Gray Fill & Border Update | Primary Border Solid, Text Color 유지 |
| **Disabled State** | Opacity (0.5), Cursor: Not-allowed | - | - |

### 2. 정보 카드 (Info Card / Feature Block)
랜딩 페이지의 핵심 콘텐츠를 모듈화하여 배치하는 블록입니다. 가독성과 구조적 전문성을 동시에 높여야 합니다.

*   **구조:** [아이콘/일러스트] + **[제목 H3]** + *[설명 텍스트 P]*
*   **배경:** White (`#FFFFFF`) 또는 Light Gray (`#F7F9FA`) (섹션에 따라 구분)
*   **Border:** Subtle Shadow Effect (Box-shadow: 0 4px 12px rgba(0, 0, 0, 0.05))를 사용하여 분리감을 부여합니다.
*   **여백(Spacing):** 내부 Padding은 상하좌우 최소 32px를 확보하여 답답함을 해소합니다.

### 3. 섹션 타이포그래피 (Section Typography)
섹션을 구분하고 콘텐츠의 계층 구조를 명확히 하는 요소입니다.

*   **H1 (메인 헤드라인):**
    *   크기: 48px ~ 60px (모바일 최적화 시 최대 32px로 축소). [근거: 모바일 환경에서의 가독성 최우선 고려]
    *   폰트: 본문용 타이포그래피 (예: Pretendard Bold)
    *   색상: Neutral-Dark (`#2C3E50`)
    *   적용 원칙: 여백(Spacing)을 매우 크게 사용하여 시각적 무게감을 줍니다.

*   **H2 (섹션 제목):**
    *   크기: 32px ~ 40px
    *   폰트: 본문용 타이포그래피 (예: Pretendard SemiBold)
    *   색상: Primary Blue (`#1A4798`) 또는 Neutral-Dark (`#2C3E50`). (강조에 따라 색 변경)
